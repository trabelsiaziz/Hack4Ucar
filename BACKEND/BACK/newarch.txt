

NEW REDUCED ARCHITECTURE — KEEP FULL MUST-HAVE FIELDS, REDUCE ENTITIES ONLY

=============================================================================

 

Goal

----

This version is intentionally REDUCED at the ENTITY level, not at the FIELD level.

The idea is:

- keep only the necessary entities/components;

- keep all must-have fields for each kept object;

- keep the DB links explicit and clean;

- keep the Java backend architecture simple and maintainable.

 

Reduction rule applied

----------------------

Removed optional / non-essential entities to keep the architecture focused:

- Campus

- Location

- RawDocumentPageRecord

- RawSheetRecord

- RawTabularRowRecord

- ResearchFact

- PartnershipFact

- InfrastructureFact

- EsgFact

- KpiSourceMapping

- KpiObservationComponent

- AlertEvent

- AnomalyEvidence

- RecommendationLink

 

Kept only the necessary entities for:

- ingestion / provenance

- organization and users

- academic / finance / HR facts

- canonical cross-service exchange

- KPI engine

- AI analysis / recommendation

- audit / orchestration

 

 

FINAL LAYERING ORDER

--------------------


L2. Access + core domain master entities

L3. Domain fact entities

L4. Canonical cross-service entity

L5. KPI governance entities

L6. KPI result / monitoring entities

L7. AI analysis entities
